## 🚫 스터디 규칙

1. 각자 영문 이니셜(Ex. 홍길동 → hgd) 브랜치에 문제 풀이 후 Push
2. PR 요청 (Reviewers, Assignees, Labels 설정)
3. 코드 리뷰는 토요일 21시 라이브 코테 전까지 하기
4. 라이브 코테 진행 이후에 주차별 문제 PR 브랜치 → main 브랜치로 병합할 예정

## :octocat: 커밋 컨벤션

- `Feat`: 새로운 문제 풀이 추가
- `Refactor`: 코드 리뷰로 인한 재풀이
- `Fix`: 풀이 수정
- `Comment`: 주석 추가
- `Rename`: 파일, 폴더명 수정
- `Remove`: 파일, 폴더 삭제
- `Docs`: Readme 수정(필요한 경우만)
